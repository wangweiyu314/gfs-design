1. 设计概述

1.1 设计原则

广发中后台系统采用专业、稳重、高效的视觉风格，核心原则：

  • 专业性：采用金融行业标准配色，体现专业可靠
  • 清晰性：信息层级分明，数据展示清晰
  • 效率性：优化操作流程，提升工作效率
  • 一致性：统一的视觉语言和交互模式

1.2 设计目标

  • 为金融中后台业务提供统一的设计标准
  • 确保不同模块间的视觉一致性
  • 提升用户操作效率和体验
  • 降低开发和维护成本

space-14px--space-1: 4px紧凑间距（图标与文字间）
space-28px--space-2: 8px小间距（内边距紧凑）
space-312px--space-3: 12px中间距（输入框内边距）
space-416px--space-4: 16px默认间距（卡片内边距）
space-520px--space-5: 20px中间距

space-1248px--space-12: 48px页面间距
space-1664px--space-16: 64px大页面间距

4.2 布局规范

页面布局

  • 页面宽度：1440px（标准桌面端）
  • 页面高度：自适应，最小 900px
  • 内容区宽度：自适应，最大 1200px
  • 侧边栏宽度：200px（展开）/ 64px（折叠）

栅格系统

  • 栅格列数：24列
  • 列宽：自适应
  • 列间距：16px（默认）/ 24px（宽松）
  • 边距：24px（默认）/ 32px（宽松）

卡片布局

  • 卡片内边距：16px（紧凑）/ 24px（默认）
  • 卡片圆角：4px（默认，边角-2）
  • 卡片阴影：0 2px 8px rgba(0, 0, 0, 0.08)

5. 阴影与边角

5.1 阴影系统

系统定义了三层阴影，每层包含三个方向（下方、左侧、右侧）：

下层投影（Low Shadow）

方向 CSS 值 使用场景
下方 0px 2px 8px rgba(0, 0, 0, 0.10) 组件 hover 状态，例如表格行和树拖动
左侧 -4px 0px 8px rgba(0, 0, 0, 0.10) 右侧面板悬停
右侧 4px 0px 8px rgba(0, 0, 0, 0.10) 左侧面板悬停

中层投影（Medium Shadow）

方向 CSS 值 使用场景
下方 0px 6px 16px rgba(0, 0, 0, 0.12) 下拉组件，例如下拉菜单 / 气泡确认框 / 选择器
左侧 -8px 0px 16px rgba(0, 0, 0, 0.12) 右侧抽屉
右侧 8px 0px 16px rgba(0, 0, 0, 0.12) 左侧抽屉

上层投影（High Shadow）

方向 CSS 值 使用场景
下方 0px 12px 32px rgba(0, 0, 0, 0.14) 警示或弹窗组件，例如全局提示 / 消息通知
左侧 -12px 0px 32px rgba(0, 0, 0, 0.14) 全屏抽屉（右侧）
右侧 12px 0px 32px rgba(0, 0, 0, 0.14) 全屏抽屉（左侧）

5.2 边角系统

系统仅使用两种圆角值：

名称 数值 使用场景
边角-1 2px 按钮
边角-2 4px 提示框、卡片、输入框、下拉面板

使用规则：按钮统一使用 2px 圆角；输入框、卡片、弹窗、下拉面板统一使用 4px 圆角；不使用更大的圆角值。

6.4 图标使用规则

  • 同一界面中保持图标风格一致（线型或填充，不可混用）
  • 图标与文字配合使用时，间距为 8px (space-2)
  • 图标按钮最小点击区域为 32px × 32px
  • 禁用状态的图标透明度为 40%
  • SVG图标需设置 overflow: visible 防止内容被裁剪
  • SVG图标线条宽度建议使用 1.5px，避免在16x16尺寸下显示过粗

6.5 使用方式

所有图标已内嵌在本文档中，可直接通过以下方式引用：

<svg width="16" height="16"><use href="#icons-common-home"/></svg>
<svg width="16" height="16"><use href="#icons-status-success-fill"/></svg>
<svg width="16" height="16"><use href="#icons-file-pdf-file"/></svg>

图标替代原则：如果在图标库中找不到所需图标，请搜索并使用同类型的图标进行替代。选择替代图标时，应确保其视觉风格、线条粗细和语义表达与目标场景保持一致。

11. 业务组件规范

11.1 业务功能表格

概述

  1. 表格整体为上下结构，阅读顺序呈"F"路线
  2. 由于数据的复杂性，采用固定列宽+可伸缩列，在其固定宽度基础上，让用户可以自由拖动列宽边缘以调整列宽
  3. 在基础表格上增加竖线以定义列宽（表格如果列之间过于紧密，也建议保留竖线，将不同数据区分开来，以提升浏览效率）

列宽设置规则

类型 说明 示例
固定列宽 对于长度有明确的数据，按字符长度设置 姓名（3-4个中文）、手机号（11位）、客户编号（11位）
自适应列宽 根据屏幕宽度等分自适应 其他不确定长度的字段

列宽数值规范：取数值最好是8的倍数，如：80px、120px、240px、400px

对齐方式

内容类型 对齐方式 说明
标题和内容左对齐更高效的阅读顺序
数字（价格、金额）右对齐方便比较，长短不一的数据
表头分组居中分组表头采用带边框表格，标题居中

斑马纹规范

横斑马纹

行极其长、数据多、需要看清楚行与行之间数据的关系时使用

  • 表头颜色：#F7F8FA
  • 表内颜色：#FAFCFF / #FFFFFF
  • 样式应当尽量轻盈，不要抢视觉，突出表内容

竖斑马纹

表明一组数据来自特定来源；或表头分组时分组比较多，看不到组的标题

  • 表头颜色：#F0F2F5
  • 表内颜色：#FAFCFF
  • 基于信息降噪原则，斑马纹不宜过于显眼

批量操作规范

  • 勾选批量按钮后，工具栏下显示选中项，可取消选择
  • 工具栏由左右两边向中间权重降低，且仅有唯一主按钮（根据需求定）
  • 表格的操作按钮放右边，未选择时hover按钮文字提示"请选择对象"，业务按钮放左边

表格数字可视化

  1. 数字带有小数的，采用右对齐，有助于提升浏览效率
  2. 如果一行数据过于多，则建议采用斑马纹，以区分各组数据
  3. 融合数值条形图的表格，正负数值采用红色与绿色加以区分
  4. 条形图对齐方式跟随数字
数值类型 浅色 深色
正数#E89696（浅红）#D12B2B（深红）
负数#A8DCB1（浅绿）#2CAA41（深绿）

图表卡片

图表项 规格
图表卡片 白底 #FFFFFF，默认内边距 20px；标准图表卡片约 613 × 390px，大型组合卡可扩展到 688px 宽；卡片圆角在可视化组件中可为 12px，嵌入业务面板时仍遵循业务卡片圆角规则。

图例 图例标记 12px × 12px，圆形或线段标记；图例文字 12px / 20px。顶部图例与标题同一行右对齐时，图例组间距 16px，标记与文字间距 4px；图例位于图表上方或下方时，与图表区域间距统一为 16px。
指标区 指标名称 16px / 24px / #4E5969；关键数值 36px / 44px / 500 / #1D2129；单位 12px / 20px / #86909C；涨跌用 #F53F3F、#00B42A，箭头 16px。

图表类型规范

图表类型 规范
环形图 常规直径 220px，小卡直径 162px；中心指标 20px / 28px / 500 / #1D2129；多系列环形图使用图表序列色，底部横向图例可换行但不可遮挡图。
柱形图 单系列柱默认使用 #165DFF 或当前 chart-series-1；分组/堆叠柱按序列色递增。X/Y 轴标签保持 12px，柱间距随容器自适应，标准图形区约 573 × 303px。
折线图 / 面积图 折线默认使用 chart-series-1，线宽约 2px；平滑面积图可使用同色渐变填充，填充透明度渐隐，不压过网格和标签。
双轴图 用于量纲不同的趋势对比，左轴与右轴都必须标注单位；柱、线、左右轴图例必须一一对应，避免只用颜色表达语义。
折柱混合图 柱表示规模/总量，线表示比例/趋势；柱优先使用 chart-series-1，线优先使用 chart-series-2，图例中线型和柱形标记要区分。

业务图表版式规则

业务图表项 规范
图表小模块 嵌入数据展示页时使用业务小模块样式：白底 #FFFFFF，圆角 4px，边框 1px solid #F0F2F5 或 #E6EAF0，内部 padding 16px。不要沿用展示型图表组件的 12px 大圆角或过大的装饰留白。
小模块标题 业务卡片内图表标题使用 16px / 24px / 500 / #313133，左对齐；如是大分组标题可加 Block Header 蓝色竖线，图表卡片内部的弱标题不加竖线。标题到图表绘图区固定 16px。
标题右侧信息 日期、机构、币种、维度切换等控件放在标题行右侧，与标题垂直居中；标题与右侧控件之间至少 16px，控件高度优先 32px。右侧说明文字如"日期：2023年2月30日"使用 12px / 20px / #67696E 或 #939599。
图表绘图区 绘图区应占满卡片剩余宽度，左右内缩与卡片 padding 对齐；Y 轴单位放在绘图区左上，右轴单位放右上。坐标轴标签与刻度使用 12px / 20px / #86909C，金融金额、比例、户数等单位必须展示。
图例位置 折线、柱形、面积图图例优先放在图表下方居中或标题行右侧；环图图例优先放在图下方，单项横向排列，标记与文字间距 4px，项间距 16px，多列时保持左对齐。
注释与问号图标 字段解释或口径说明使用 14px 问号图标，颜色 #C8C9CC，紧贴字段标题或中心标签右侧 4px；不要把注释图标悬浮在图形区内部。注释内容使用 Tooltip，复杂口径改 Popover。
环形图中心 环图中心主指标使用数字字体，20px / 28px / 500 / #1D2129；单位与指标同基线或紧跟数字；中心标签使用 12px / 20px / #67696E，问号图标位于标签右侧。中心内容必须水平垂直居中，不随图例挤压偏移。
环形图布局 双环图并列时左右卡片间距 16px；单个环图与图例形成垂直结构，环图上方不额外加大空白。常规环图直径 140px-162px 适合业务卡片，展示型大图可用 220px。
多图组合 同一卡片内左右并列图使用 16px 间距；左图偏结构、右图偏趋势时，二者顶部标题或绘图区顶部对齐。上下图表模块之间保持 32px，避免连续图表粘连。
对齐方式 标题、单位、Y 轴标签、图例文本均左对齐或按图表语义居中；金融数值、金额、比例在表格和指标卡中右对齐，图表轴标签保持系统默认方向，不倾斜到影响阅读。
空态与加载 图表无数据时保留标题和筛选条件，绘图区显示空态图标与"暂无数据"，空态区域高度不低于 160px；加载态覆盖绘图区，不覆盖标题行操作。

筛选区规范

  • 筛选区域可以加上底色 #F7F8FA，也可以不加，视情况而定，不做强制规定
  • 单行筛选项根据页面宽度自适应数量，默认一行3个，按钮不换行
  • 如需要考虑使用快捷填充操作加上时间筛选项这种超长输入框时需要考虑一个选项单独占一行
  • 输入框长度视实际情况，按8的倍数设置
  • 默认最多显示3行筛选项，当超过3行则使用收起，筛选、查询按钮移到最下方，点击展开显示所有筛选项
  • 分辨率≥1280时，每行展示3列；分辨率≥768时，每行展示2列

12.3 表单布局 (Form Layouts)

12.3.1 左对齐标签（左右结构）

适用于希望用户放慢速度、仔细思考表单中每个表单项的场景，如含有大量可选输入框或高级设置的陌生数据时；或者是详情页陈列数据。

左对齐标签表单

12.3.2 右对齐标签（左右结构）- 推荐

右对齐标签与表单域联系紧密，便于用户填写，操作效率高。常规表单场景推荐使用右对齐标签。

右对齐标签表单（推荐）

12.3.3 顶对齐标签（上下结构）

顶对齐标签对齐舒适，用户视线固定，视觉动线一直向下，有清晰的操作路径。标签长度超过5个中文字符时，建议所有表单采用顶对齐。
