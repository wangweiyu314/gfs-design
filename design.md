## 1. 设计概述

### 1.1 设计原则

广发中后台系统采用专业、稳重、高效的视觉风格，核心原则：

- **专业性**：采用金融行业标准配色，体现专业可靠
- **清晰性**：信息层级分明，数据展示清晰
- **效率性**：优化操作流程，提升工作效率
- **一致性**：统一的视觉语言和交互模式

### 1.2 设计目标

- 为金融中后台业务提供统一的设计标准
- 确保不同模块间的视觉一致性
- 提升用户操作效率和体验
- 降低开发和维护成本

### 1.3 适用范围

本设计规范适用于广发证券中后台系统的所有界面设计，包括：

- 证券借入管理
- 专项融券
- 券池管理
- 运营风控
- 绩效管理
- 统计分析
- 系统管理

## 3. 排版系统

### 3.1 字体规范

#### 中文字体

- **主字体**：微软雅黑 (Microsoft YaHei)
- **备用字体**：PingFang SC, sans-serif
- **CSS 写法**：`font-family: 'Microsoft YaHei', 'PingFang SC', sans-serif;`

#### 英文字体

- **主字体**：SF Pro Display, -apple-system
- **备用字体**：Helvetica Neue, Arial, sans-serif
- **CSS 写法**：`font-family: 'SF Pro Display', -apple-system, 'Helvetica Neue', Arial, sans-serif;`

#### 数字字体

- **主字体**：DIN Alternate, DIN
- **备用字体**：SF Pro Display, Helvetica Neue
- **CSS 写法**：`font-family: 'DIN Alternate', 'DIN', 'SF Pro Display', 'Helvetica Neue', sans-serif;`

#### 完整字体栈

```css
font-family: 'Microsoft YaHei', 'PingFang SC',
             'SF Pro Display', 'DIN Alternate',
             -apple-system, 'Helvetica Neue', Arial, sans-serif;
```

## 5. 阴影与边角

### 5.1 阴影系统

系统定义了三层阴影，每层包含三个方向（下方、左侧、右侧）：

#### 下层投影（Low Shadow）

| 方向 | CSS 值                              | 使用场景                  |
| -- | ---------------------------------- | --------------------- |
| 下方 | `0px 2px 8px rgba(0, 0, 0, 0.10)`  | 组件 hover 状态，例如表格行和树拖动 |
| 左侧 | `-4px 0px 8px rgba(0, 0, 0, 0.10)` | 右侧面板悬停                |
| 右侧 | `4px 0px 8px rgba(0, 0, 0, 0.10)`  | 左侧面板悬停                |

#### 中层投影（Medium Shadow）

| 方向 | CSS 值                               | 使用场景                      |
| -- | ----------------------------------- | ------------------------- |

### 6.3 图标使用规则

- 同一界面中保持图标风格一致（线型或填充，不可混用）
- 图标与文字配合使用时，间距为 8px (space-2)
- 图标按钮最小点击区域为 32px × 32px
- 禁用状态的图标透明度为 40%
- **SVG图标需设置** **`overflow: visible`** **防止内容被裁剪**

**CSS 示例**：

```css
/* 方式1：包装元素包含SVG */
.icon {
  width: 16px;
  height: 16px;
  display: inline-flex;
  align-items: center;
  justify-content: center;
  flex-shrink: 0;
}

.icon svg {
  width: 100%;
  height: 100%;
  fill: currentColor;
  overflow: visible;  /* 防止图标被裁剪 */
}

/* 方式2：SVG直接应用.icon类 */
svg.icon {
  width: 16px;
  height: 16px;
  display: inline-block;
  vertical-align: middle;
  overflow: visible;
}

/* 确保SVG图标线条清晰 */
svg.icon, .icon svg {
  shape-rendering: geometricPrecision;
}
```

- 图标颜色跟随文字颜色，使用 `currentColor` 继承
- **SVG图标线条宽度建议使用 1.5px，避免在16x16尺寸下显示过粗**
- **信息图标示例**：

```html
<svg class="alert-icon icon" viewBox="0 0 24 24" fill="none" stroke="#2A6CDD" stroke-width="1.5">
  <circle cx="12" cy="12" r="10"/>
  <line x1="12" y1="16" x2="12" y2="12" stroke-linecap="round"/>
  <circle cx="12" cy="8" r="1" fill="#2A6CDD" stroke="none"/>
</svg>
```

- SVG 图标引用方式：

```html
<!-- 推荐：内联 SVG，无需外部文件依赖 -->
<svg width="16" height="16" viewBox="0 0 16 16" fill="none" xmlns="http://www.w3.org/2000/svg">
  <path d="..." fill="#313133"/>
</svg>
```

> **注意**：所有示例代码中的图标已内联为 SVG，无需额外 `icons/` 目录即可正常显示。

## 11. 业务组件规范

> 以下内容来自《业务组件.pdf》，详细说明了业务场景中的表格、审批流程、组织架构树等组件的设计规范。

### 11.1 业务功能表格

#### 概述

1. 表格整体为上下结构，阅读顺序呈"F"路线
2. 由于数据的复杂性，采用**固定列宽+可伸缩列**，在其固定宽度基础上，让用户可以自由拖动列宽边缘以调整列宽
3. 在基础表格上增加竖线以定义列宽（表格如果列之间过于紧密，也建议保留竖线，将不同数据区分开来，以提升浏览效率）

#### 列宽设置规则

| 类型    | 说明                 | 示例                            |
| ----- | ------------------ | ----------------------------- |
| 固定列宽  | 对于长度有明确的数据，按字符长度设置 | 姓名（3-4个中文）、手机号（11位）、客户编号（11位） |
| 自适应列宽 | 根据屏幕宽度等分自适应        | 其他不确定长度的字段                    |

**列宽数值规范**：取数值最好是8的倍数，如：80px、120px、240px、400px

#### 对齐方式

| 内容类型      | 对齐方式 | 说明               |
| --------- | ---- | ---------------- |
| 标题和内容     | 左对齐  | 更高效的阅读顺序         |
| 数字（价格、金额） | 右对齐  | 方便比较，长短不一的数据     |
| 表头分组      | 居中   | 分组表头采用带边框表格，标题居中 |

#### 斑马纹规范

**横斑马纹**：行极其长、数据多、需要看清楚行与行之间数据的关系时使用

- 表头颜色：`#F7F8FA`
- 表内颜色：`#FAFCFF` / `#FFFFFF`
- 样式应当尽量轻盈，不要抢视觉，突出表内容

**竖斑马纹**：表明一组数据来自特定来源（如wind，会定时更新）；或表头分组时分组比较多，看不到组的标题

- 表头颜色：`#F0F2F5`
- 表内颜色：`#FAFCFF`
- 基于信息降噪原则，斑马纹不宜过于显眼，建议有微微区分即可

#### 批量操作规范

- 勾选批量按钮后，工具栏下显示选中项，可取消选择
- 多选框规范具体规则见5.3多选框规范
- 工具栏由左右两边向中间权重降低，且仅有唯一主按钮（根据需求定）
- 表格的操作按钮放右边，未选择时hover按钮文字提示"请选择对象"，业务按钮放左边

**示例**：

```
已选择 6 项  [取消选择]
[刷新数据] [导入数据] [自定义列] [编辑模式] [复制数据] [导出对象] [删除]
```

#### 自定义列设置

- 点击"自定义列"按钮，显示设置弹窗
- 可以对表头字段重命名（对应的后端属性不变）
- 参照5.15穿梭框规范

**设置弹窗内容**：

- 当前选定的字段列表
- 可选字段列表
- 操作按钮：载入配置、保存当前配置、恢复默认、确认、取消

#### 表头筛选

- 点击表头字段筛选按钮，弹出多/单选对话框
- 筛选状态显示：全部、博士、硕士、本科等选项
- 操作按钮：确定、重置

#### 表头说明

- 表头表述尽量精简
- 对于比较复杂的表头，可以定义一个专有名词
- 鼠标 hover 弹出 tooltips 解释说明

#### 单元格交互

**单元格链接**：

- 单元格内可以存在文字链接
- 点击文字链接进入对应流程

**可编辑单元格**：

- 单元格显示编辑按钮则表示此单元格可编辑
- 点击图标激活编辑框
- 列宽最好预留输入框宽度
- 点击非编辑区或退出并保存编辑

**可编辑对话框表单**：

- 有标签长度超过5个中文字符时，建议所有表单采用顶对齐（上下结构），以节约横向空间

#### 表格数字可视化

1. 数字带有小数的，采用右对齐，有助于提升浏览效率
2. 如果一行数据过于多，则建议采用斑马纹，以区分各组数据
3. 融合数值条形图的表格，正负数值采用红色与绿色加以区分
4. 条形图对齐方式跟随数字

**颜色规范**：

- 正数：`#E89696`（浅红）/ `#D12B2B`（深红）
- 负数：`#A8DCB1`（浅绿）/ `#2CAA41`（深绿）

### 11.2 审批流程

#### 流程日志

- 流程日志占屏幕右侧1/4
- 节点之间通过竖线连接
- 若为最早一项，则圆点下方不再显示竖线
- 状态颜色：
  - 进行中：蓝色
  - 已完成：绿色
  - 未通过/拒绝：红色

#### 流程详情弹窗

- 内容最多显示2行，超过部分显示"..."
- 鼠标 hover 弹出 tooltips 解释说明
- 参考6.10手风琴面板的交互，每次只展开一个详情

**弹窗内容**：

- 处理节点
- 处理人
- 开始时间
- 结束时间
- 处理结果
- 说明
- 操作（查看详情）

#### 提交审核弹窗

- 节点处理人模块为穿梭框变体
- 与原有的穿梭表有所差异
- 包含备注输入框（暗提示，限制100字符）
- 操作按钮：取消、拒绝、通过、提交审核

#### 文字链接按钮

「查看详情」使用文字链接按钮：

- 比例：75% / 25%
- 背景：`N2-40`
- 文字：`B5-10`
- 状态：Normal、Hover、Active、Disable

### 11.3 部门、公司人员架构树

#### 单选部门

- 适用公司人员组织架构树单选部门
- 包含搜索功能：请输入处理人姓名
- 显示推送范围、常用范围

**状态说明**：

- 悬停状态
- 选中状态
- 常规状态

#### 多选部门

- 选择部分子集时，选中列表显示子集
- 选中全部子级时，选中列表显示父级
- 当父级存在本部时候，显示"全辖"与"本部"按钮

**选中规则**：

- **选中全辖时**：选中部分子级，选中列表显示此子级；选中全部子级时，选中列表显示全辖
- **选中本部时**：选中列表显示本部与子级
- **勾选子级**：显示具体选中的子级
- **全部选中**：显示父级名称

#### 保存配置流程

- 适用关联业务、发送业务
- 功能按钮：添加发送人、保存当前名单、载入名单

**保存配置弹窗**：

- 输入名单名称
- 操作按钮：取消、保存

**载入配置**：

- 显示已有配置列表
- 支持编辑已有配置名称（原位编辑，回车确定）
- 悬停状态显示编辑按钮

#### 弹窗尺寸规范

多选弹窗分两种尺寸：

- **小尺寸**：宽720px，高600px（对应弹窗规范的大尺寸）
- **大尺寸**：宽900px，高600px（对应弹窗规范的特大尺寸）

**高度限制**：

- 最小高度：160px
- 最大高度：600px
